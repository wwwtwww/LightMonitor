# UI 现代化重构计划 (LightMonitor)

本计划旨在将 LightMonitor 升级为现代化的暗色监控大盘风格（类似 Grafana），提升专业感与可读性。

## 1. 核心设计风格
- **主题**：深色模式 (Dark Mode)，以深灰/黑背景为主，辅以高亮强调色。
- **布局**：卡片式 (Card-based)，使用 Grid 布局，增加留白。
- **字体**：Inter / Roboto / System UI，强调数字的可读性。
- **图表**：去除多余边框，使用深色底，高亮线条与填充。

## 2. 关键组件改造

### 2.1 全局样式 (styles.css)
- 引入 CSS 变量定义颜色系统（背景、前景、边框、强调色）。
- 重置默认样式，应用深色背景 `#181b1f`。
- 定义卡片组件 `.card`：深色背景 `#22252b`，微圆角，无边框或极细边框。

### 2.2 首页 (index.html)
- **Header**: 增加品牌感，右侧增加“新增”按钮的显著性。
- **列表**:
  - 将表格 (Table) 改造为 **数据卡片网格** (Data Grid) 或 **现代列表**。
  - 每个数据库展示为一个卡片：
    - 顶部：库名 + 类型图标 + 在线状态点。
    - 中部：关键指标 (QPS, TPS, 慢查) 的实时数值。
    - 底部：操作栏 (测试、详情、管理)。

### 2.3 详情页 (details.html)
- **顶部统计行 (Stat Cards)**:
  - 在图表上方新增一行 4 个大卡片。
  - 展示：当前会话数、QPS、TPS、今日慢查数。
  - 样式：大号数字 + 标签 + 趋势指示（如由绿色/红色箭头表示，或仅展示当前值）。
- **图表区域**:
  - 保持 2x2 布局，但增加卡片内边距。
  - Chart.js 配置更新：适配深色模式（坐标轴颜色、网格线颜色变淡）。
- **诊断面板**:
  - 锁列表与慢 SQL 列表样式优化，适配深色背景。

## 3. 实施步骤

### 阶段一：基础设施 (Infrastructure)
1.  **CSS 变量重构**: 在 `styles.css` `:root` 中定义深色调色板。
2.  **基础组件样式**: 更新 body, button, input, modal 的深色样式。

### 阶段二：首页改造 (Home Page)
1.  **结构调整**: 修改 `index.html`，将表格容器改为 Grid 容器。
2.  **渲染逻辑**: 修改 `app.js` 的 `renderList`，生成卡片 DOM 而非 `<tr>`。

### 阶段三：详情页改造 (Detail Page)
1.  **布局调整**: 修改 `details.html`，插入 Stat Cards 区域。
2.  **样式适配**: 更新图表容器样式。
3.  **图表配置**: 修改 `details.js` 中的 Chart 配置，注入深色主题颜色（Grid lines, Ticks）。

### 阶段四：细节打磨
1.  **状态图标**: 优化在线/离线指示器。
2.  **交互反馈**: 按钮 Hover 态，卡片 Hover 态。
3.  **响应式**: 确保在小屏幕下卡片自动堆叠。

## 4. 预览效果预期
- **背景**: 深灰 (`#111217`)
- **卡片**: 稍亮灰 (`#181b1f`)
- **文字**: 灰白 (`#dce0e5`) / 辅助灰 (`#6e7681`)
- **图表**: 蓝色线条 (`#3b82f6`)，红色告警 (`#ef4444`)
